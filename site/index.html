Drive your PrusaLink printers
from any web app you trust

A browser extension and a tiny client library that let a web app send G-code to your own PrusaLink printers over the LAN — gated by an explicit, per-origin consent prompt. Printer URLs and credentials live only in the extension; the web app never sees them.

How it works

Browser-mediated, consent-gated, credential-isolated.
🔐

Your credentials stay put

Printer addresses and API keys live only in the extension. A page learns nothing but a name, model, and status — never a URL or secret.

✅

You approve each origin

First contact from a site triggers a consent prompt showing the browser-verified origin. Printing defaults to a per-print confirmation you can disable.

🖨️

Any app, one tiny shim

Apps depend only on a zero-dependency npm package and talk over postMessage. No allowlist — your consent is the gate.

Install

Two pieces: the extension, and (for app authors) the shim.
  1. Get the extension. Store listings are on the way. For now, grab a build from Releases (or build from source) and load it unpacked (chrome://extensions → Developer mode → Load unpacked).
  2. Add your printer in the extension's options: its LAN address and an API key (or Digest, or none behind a trusted proxy). You grant each host permission at add-time.
  3. App authors: depend on the shim — npm i @tibordp/prusalink-bridge — and call createBridge(). See the client docs.
